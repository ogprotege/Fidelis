STEP 10 — iOS SANITY, PUSH, MERGE  (no Claude needed; this is you + Xcode)
────────────────────────────────────────────────────────────────────

10.0  PREREQUISITES (one-time)
  - A Mac with Xcode 15 or newer: App Store ▸ search "Xcode" ▸ Get.
    First launch asks to install extra components — accept.
  - Then in Terminal:
      xcode-select --install        (command-line tools, if prompted)
  - A free Apple ID is enough for the iPhone *simulator*. Running on
    your real iPhone needs that Apple ID added in Xcode ▸ Settings ▸
    Accounts (free "Personal Team" works; apps expire after 7 days,
    fine for testing).

10.1  MERGE THE OPEN PR FIRST
  - Merge PR #5 (Quote of the Day, v1.2.0) on GitHub:
      https://github.com/ogprotege/Fidelis/pull/5
  - Then locally:
      cd /Users/biscuit/Fidelis/app
      git checkout main
      git pull origin main
      npm install
      npm test                      (expect: all checks passed, manifest verified)

10.2  BUILD THE WEB APP INTO THE iOS SHELL
      npm run build
      npx cap sync ios
      npx cap open ios              (opens ios/App in Xcode)

10.3  RUN IN THE SIMULATOR
  - Top toolbar in Xcode: scheme should say "App"; next to it pick a
    simulator, e.g. "iPhone 16 Pro". Press ⌘R (or the ▶ button).
  - First simulator boot is slow; let it finish.
  - SPOT-CHECK, against the same list as Step 9:
      Today tab: exactly five cards — Verse, Quote, Today in the
        Church (with the Angelus line), Rosary, Continue Reading
      Mass: the readings render, date arrows work, region select works
      Read: open John 1 in all three translations; parallel view
      Night mode toggle; tap a verse → action bar; add a note
  - The web checks from Step 9 (Good Friday 2016 etc.) all apply here
    too if you want to repeat them — it is the same engine.

10.4  THE HOME-SCREEN WIDGET (one-time target creation — Xcode only,
      this genuinely cannot be scripted; ~5 minutes)
  1. Xcode: File ▸ New ▸ Target… ▸ search "Widget Extension" ▸ Next.
  2. Product Name: FidelisWidget — UNTICK "Include Configuration
     App Intent" — Finish — "Activate scheme?" → Activate.
  3. In the file list, delete the FidelisWidget folder's template
     .swift file Xcode just made (Move to Trash).
  4. Drag from Finder into the FidelisWidget group in Xcode:
       ios/WidgetExtension/FidelisWidget.swift
       ios/WidgetExtension/votd.json
     In the dialog: tick "Copy items if needed" and Target membership
     = FidelisWidget (NOT App).
  5. Pick the FidelisWidget scheme ▸ ⌘R. The simulator shows the
     widget preview. Verify it shows TODAY'S verse — the same one the
     web Today page shows. That equality is the whole point (P1-9).

10.5  ON YOUR REAL IPHONE (optional now, needed eventually)
  - Plug the phone in (or Wi-Fi pairing), pick it as the run target.
  - Xcode will demand signing: project navigator ▸ App target ▸
    Signing & Capabilities ▸ Team = your Apple ID; change the Bundle
    Identifier to something unique, e.g. app.fidelis.bible.biscuit.
    Do the same for the FidelisWidget target.
  - First run: iPhone Settings ▸ General ▸ VPN & Device Management ▸
    trust your developer certificate. Run again.

10.6  IF SOMETHING BREAKS
  - "Unable to boot simulator" → Xcode ▸ Settings ▸ Platforms ▸
    download the iOS runtime.
  - Blank white screen in the app → you forgot npm run build or
    npx cap sync ios; re-run both, then ⌘R.
  - Widget shows placeholder text → votd.json missing from the
    FidelisWidget target's Copy Bundle Resources (step 10.4.4).
